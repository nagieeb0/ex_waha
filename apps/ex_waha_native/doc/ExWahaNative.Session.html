    ExWahaNative.Session (ExWahaNative v0.2.0)

    Copy Markdown View Source

    Per-WhatsApp-account state machine.

    Holds the account, current status, and a set of subscribers (LiveViews, PubSub bridges, custom handlers) that get {:ex_waha_native, kind, payload} messages on every event from the Go bridge for this session.

    Outbound calls are routed through ExWahaNative.Bridge; this GenServer doesn't serialize them — the Bridge does.

    qr(name)

    @spec qr(String.t() | ExWahaNative.Account.t()) :: String.t() | nil

    Returns the most recent QR code seen for the session, or nil if none.

    When the account is unpaired, the bridge emits successive qr events as the underlying WhatsApp QR rotates every ~20s. This helper exposes the latest one so a controller/LiveView can render it without subscribing.

    request_pairing_code(name, phone)

    @spec request_pairing_code(String.t() | ExWahaNative.Account.t(), String.t()) ::
  {:ok, String.t()} | {:error, term()}

    Request a phone-pairing code instead of QR.

    WhatsApp 2.24+ supports linking a device via a numeric pairing code shown on the phone. Returns {:ok, code} (formatted as e.g. "ABCD-EFGH") or an {:error, reason} if the session is not in a state to pair.
